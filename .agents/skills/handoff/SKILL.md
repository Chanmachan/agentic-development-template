# Handoff Prompt 生成

`/clear` の直前に実行し、次セッションがコンテキストなしで作業を再開できる **自己完結した引き継ぎプロンプト** を生成する。

## 目的

長いセッションで膨らんだコンテキストを `/clear` で破棄する前に、「次に何をどう進めるか」を 1 つのプロンプトに凝縮する。次セッションはこのプロンプトを貼るだけで、必読ファイルを読み直して作業を継続できる状態にする。

## 引数

$ARGUMENTS は任意。次セッションで最初にやることを一言で渡せる (例: `G-1 から着手`)。空なら現在の作業文脈から推測する。

## 手順

1. 現在の git 状態を確認する (並列実行):
   - `git branch --show-current` / `git status --short` / `git log --oneline -5`
2. 今セッションで何が確定・完了したかを整理する:
   - 合意した設計判断、作成・更新したファイル (特に `tasks/<id>-todo.md` と `tasks/tasks.jsonl` の状態行)、未解決の論点
   - **会話文脈にしかない情報** (口頭合意・判断の根拠) を優先して拾う。コードや git log から再現できる事実は冗長に書かない
3. 次にやることを特定する ($ARGUMENTS があればそれを起点に):
   - 次の最小作業単位、着手前の前提条件 (ブランチ作成など)
4. 以下の構造で **コードブロックに入れた引き継ぎプロンプト** を出力する (ユーザがそのままコピーできるように):

   ```
   ## 状況
   - ブランチ / 直近 PR / 今どのフェーズか (1-3 行)

   ## 必読 (この順で読む)
   1. tasks/tasks.jsonl — in-flight (status≠done) の一覧。現在状態の索引
   2. tasks/<id>-todo.md — <該当セクション名>。計画と確定判断の真実の源
   3. <関連 tasks/done/*.md や ADR、対象コードファイル>
   4. AGENTS.md / CLAUDE.md / .claude/rules/git.md / .codex/rules/git.md — workflow ルール

   ## 今セッションで確定したこと (会話文脈にしかない要点)
   - <設計判断や口頭合意のうち、ファイルに書ききれていない or 背景が重要なもの>

   ## 次にやること
   - <最初の作業単位。着手前の前提 (ブランチ作成など) を明示>
   - <承認ゲートや検証手順があれば明記>

   ## 注意
   - <このセッションで効いていた制約。commit ルール、no-verify 禁止、スコープ境界など>
   ```

5. プロンプト本文とは別に、`/clear` 後に貼る想定であることを 1 行で添える。

## 原則

- **自己完結**: 次セッションは今の会話を見られない前提。曖昧な指示語 ("さっきの件" 等) を使わない。
- **重複を避ける**: todo ファイルに既に書いた計画は再掲せず「読め」と指す。プロンプトには *会話にしかない* 文脈を残す。
- **簡潔に**: 必読ファイルを指せば足りるものは指すだけ。プロンプトが長文計画書になったら本末転倒。
- 引き継ぎ内容を todo ファイルに反映すべき差分があれば、プロンプト生成前にそのファイルを更新する (プロンプトは todo を指すだけにする)。

## 使い方

/handoff
/handoff G-1 から着手
